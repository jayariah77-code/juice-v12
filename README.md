## What is Juice v12?

**Juice v12** is a fully standalone WhatsApp Multi-Device Bot built on **gifted-baileys**. With **766 commands** covering AI chat, media downloads, group management, games, stickers, football stats, font effects and more — it runs on any panel, VPS, or cloud platform with a single command.

> No database server &nbsp;·&nbsp; No Redis &nbsp;·&nbsp; No pm2 &nbsp;·&nbsp; No build step &nbsp;·&nbsp; Just `node index.js`

## Quick Start

**1. Clone**
```bash
git clone https://github.com/jayariah77-code/juice-v12.git
cd juice-v12
```

**2. Install**
```bash
npm install --legacy-peer-deps
```

**3. Configure** — edit `.env`
```env
OWNER_NUMBER=254753204154        # your number — country code, no + or spaces
SESSION_ID=JUICE~paste_here      # from the Session Generator link above
```

**4. Run**
```bash
node index.js
```

> **No session yet?** Leave `SESSION_ID` blank → the bot prints a **pairing code** in the logs → open WhatsApp → Settings → Linked Devices → Link a Device → Link with phone number → enter the code.

<p align="center"><img src="https://i.imgur.com/dBaSKWF.gif" height="20" width="100%"></p>

## Session Setup

| Method | Steps |
|---|---|
| **Session Generator** *(recommended)* | Go to the [Session Generator](https://juice-v12-session-generator.onrender.com/pair) → enter your number → scan/enter the code → copy the `JUICE~...` ID → paste into `SESSION_ID` in `.env` |
| **Pairing Code** | Leave `SESSION_ID` blank → run the bot → a pairing code appears in logs → enter it in WhatsApp |
| **Migrate session** | Copy `sessions/<your-number>/` to the new host — bot auto-reconnects |

<details>
<summary><b>Heroku</b></summary>

```bash
heroku create your-bot-name
git push heroku main
heroku config:set OWNER_NUMBER=254753204154 SESSION_ID=JUICE~...
heroku ps:scale web=1
```
FFmpeg is installed automatically via the buildpack declared in `app.json`.
</details>

<details>
<summary><b>Railway</b></summary>

1. Click the Railway button above
2. Set `OWNER_NUMBER` and `SESSION_ID` in the **Variables** tab
3. `railway.toml` and `nixpacks.toml` are already configured — FFmpeg is installed automatically

</details>

<details>
<summary><b>Render</b></summary>

1. Fork this repo
2. New → **Background Worker** → connect your fork
3. Set `OWNER_NUMBER` and `SESSION_ID` in the **Environment** tab
4. Deploy — `render.yaml` handles the rest

</details>

<details>
<summary><b>Pterodactyl / Any Panel</b></summary>

```
Startup command : node index.js
Install command : npm install --legacy-peer-deps
```
Set `OWNER_NUMBER` and `SESSION_ID` in the panel's environment / variables section.

</details>

## Architecture

```
juice-v12/
├── index.js          # Connection, session loader, auto-restart supervisor
├── client.js         # All 766 command handlers
├── setting.js        # Bot config (name, prefix, owner, toggles)
├── .env              # Your keys (OWNER_NUMBER, SESSION_ID)
├── package.json      # Dependencies
├── Procfile          # Heroku — web: node index.js
├── app.json          # Heroku one-click deploy + FFmpeg buildpack
├── render.yaml       # Render background worker config
├── railway.toml      # Railway deployment config
├── nixpacks.toml     # Railway build — installs FFmpeg
├── library/
│   ├── lib/          # myfunc.js, exif.js, store.js
│   ├── menulist/     # Menu text for each command category
│   └── scrape/       # Scrapers (Facebook, Wikipedia, news, etc.)
├── plugin/           # polination-ai.js (external plugin example)
├── database/         # leaderboard.json, users.json, tebakgame.json
├── images/           # Bot logo assets
├── sessions/         # WhatsApp auth files (auto-created, git-ignored)
└── tmp/              # Temporary media files (auto-cleaned every 15 min)
```
