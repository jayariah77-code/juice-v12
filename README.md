  ## 🤖 What is Juice v12?

  **Juice v12** is a blazing-fast, feature-rich WhatsApp Multi-Device Bot built on the **gifted-baileys** protocol. With **700+ commands** spanning AI chat, media downloads, group management, games, font effects, TextPro image art, football predictions and more — it's the only bot you'll ever need.

  > ⚡ Instant responses &nbsp;•&nbsp; 🛡️ Built for reliability &nbsp;•&nbsp; 🎨 Fully customizable &nbsp;•&nbsp; 🔌 Plugin-ready &nbsp;•&nbsp; 🌍 Multi-language

  - **Node.js** v18 or higher

  - A WhatsApp account for pairing

  ## 🚀 Installation

  ### 1. Clone the repository
  ```bash
  git clone https://github.com/jayariah77-code/juice-v12.git
  cd juice-v12
  ```

  ### 2. Install dependencies
  ```bash
  npm install
  ```

  ### 3. Configure
  ```env
  SESSION_ID=Juice v12~eyJub2lzZUtleS...   # from Session Generator
  OWNER_NUMBER=254753204154              # your WhatsApp number (no + or spaces)
  ```

  ### 4. Run
  ```bash
  node index.js
  ```

  > 💡 First time? Get your session ID instantly → **[Session Generator](https://juice-v12-session-generator.onrender.com/pair)**

<p align="center"><img src="https://i.imgur.com/dBaSKWF.gif" height="20" width="100%"></p>

  ## 🔑 Session Setup

  | Method | How |
  |---|---|
  | **Pairing Code** | Run bot → select `1` → enter phone number → enter code in WhatsApp |
  | **Session ID** | Visit [Session Generator](https://juice-v12-session-generator.onrender.com/pair) → paste `SESSION_ID` in `.env` |
  | **Existing Session** | Copy `sessions/<number>/` folder to new host — auto-reconnects |

  ## ☁️ Deploy to Cloud

  <div align="center">

  | Platform | One-Click Deploy |
  |:---:|:---:|
  | **Heroku** | [![Deploy to Heroku](https://www.herokucdn.com/deploy/button.svg)](https://heroku.com/deploy) |
  | **Railway** | [![Deploy on Railway](https://railway.app/button.svg)](https://railway.app) |
  | **Render** | [![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com) |

  </div>

  <details>
  <summary><b>📋 Heroku Setup</b></summary>

  ```bash
  heroku create your-bot-name
  heroku buildpacks:set heroku/nodejs
  git push heroku main
  heroku ps:scale worker=1
  ```
  > Uses a `worker` dyno — the bot is a persistent process, not a web server.
  </details>

  <details>
  <summary><b>📋 Railway Setup</b></summary>

  ```bash
  railway init
  railway up
  ```
  Set the start command to `node index.js` in your Railway project settings.
  </details>

  <details>
  <summary><b>📋 Render Setup</b></summary>

  1. Fork this repo
  2. Connect your fork to Render → **New Background Worker**
  3. Set start command: `node index.js`
  4. Add your `.env` variables in the **Environment** tab
  5. Click **Deploy** — the bot connects automatically

  </details>
